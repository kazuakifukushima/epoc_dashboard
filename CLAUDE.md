# CLAUDE.md

## プロジェクト概要
このリポジトリは、EPOC由来のExcelファイルを自動判定し、
可視化済みExcelに変換するためのツール群を含む。

主な対応形式:
1. 経験症候・疾患ファイル
2. 評価ファイル（研修医評価 / 指導医評価）

---

## 開発方針
- まず Python 側のパースと出力の安定性を最優先にする
- UI は最小構成から始める
- 形式揺れに強い parser を重視する
- 元データは変更せず、常に別名出力する
- ロジックは「判定」「抽出」「集計」「出力」に分離する

---

## Claude Code への依頼原則
### 1. 変更前に設計を簡潔に示す
- 何を変更するか
- どのファイルに触るか
- 互換性への影響はあるか

### 2. parser は小さく分割する
- detector
- symptom/disease parser
- evaluation parser
- formatter / writer

### 3. 破壊的変更を避ける
- 列名追加は後方互換にする
- 既存の出力シート名はできるだけ維持する

### 4. テスト可能性を確保する
- 匿名化サンプルExcelで再現できるようにする
- 関数単位で入力と出力を明確化する

### 5. エラーメッセージを実務向けにする
悪い例:
- invalid format

良い例:
- 「研修医評価」「指導医評価」シートを認識できませんでした
- 経験症候・疾患ブロックのヘッダを認識できませんでした

---

## コーディング規約
- Python は関数分割を優先
- pandas 依存は許容するが、可読性を優先
- openpyxl での装飾は最低限明確に
- TypeScript 側は server route と UI を分離
- パス依存は README に明記する
